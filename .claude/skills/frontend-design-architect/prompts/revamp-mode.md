# Revamp Mode

This mode is used when the user already has an interface and wants it improved.

The goal is to transform an existing UI into a more professional, modern, clear, and premium experience while preserving the product’s functionality.

---

# When To Use

Use this mode when the user says things like:

- “Bu UI çok amatör duruyor.”
- “Landing page’i daha premium yap.”
- “Dashboard’ımın UX’ini iyileştir.”
- “Bu tasarım eski görünüyor, modernize et.”

This mode focuses on **improving**, not replacing the product concept.

---

# Core Goal

Your job is to:

1. Identify design weaknesses
2. Explain why they reduce quality
3. Propose a stronger visual direction
4. Suggest component and layout improvements
5. Preserve the product’s core functionality

Revamp Mode should **upgrade the interface without breaking its purpose**.

---

# What To Analyze

When reviewing an existing UI, examine these areas.

## 1. Hierarchy

Check:

- headline clarity
- section priority
- visual emphasis
- CTA prominence

Questions:

- Is the main action obvious?
- Are headings readable at a glance?
- Is the hierarchy logical?

---

## 2. Spacing System

Look for:

- inconsistent margins
- cramped sections
- uneven card padding
- chaotic spacing rhythm

A clean interface often follows a consistent spacing scale (for example an 8pt system).

---

## 3. Typography

Check:

- number of fonts
- heading scale
- readability
- contrast

Common problems:

- too many font styles
- weak headline contrast
- small unreadable body text

Goal:

Create a **clear typography hierarchy**.

---

## 4. Color Usage

Analyze:

- palette consistency
- accent color usage
- contrast quality

Common issues:

- random colors
- weak CTA color
- low contrast text

Revamp should simplify color usage.

---

## 5. Component Quality

Inspect:

- buttons
- inputs
- cards
- tables
- navigation

Questions:

- Are components visually consistent?
- Are buttons clearly interactive?
- Are cards overly heavy or overly flat?

---

## 6. Layout Discipline

Look at:

- section order
- grid alignment
- column structure
- navigation placement

Common problems:

- cluttered layout
- inconsistent alignment
- broken grid structure

Revamp Mode should restore **layout discipline**.

---

## 7. Visual Noise

Detect unnecessary elements:

- excessive shadows
- random gradients
- decorative icons
- distracting backgrounds

Good design removes noise and increases clarity.

---

# Improvement Strategy

After analyzing the UI, propose improvements in these layers:

1. Hierarchy fixes
2. Typography improvements
3. Color simplification
4. Component redesign
5. Layout cleanup
6. Spacing system alignment

Focus on **systemic improvements**, not random tweaks.

---

# Output Structure

When using Revamp Mode, follow this structure.

## 1. Current UI Problems

Identify the main weaknesses.

Examples:

- weak visual hierarchy
- inconsistent spacing
- confusing navigation
- outdated component style

---

## 2. New Visual Direction

Explain how the improved design should feel.

Examples:

- cleaner hierarchy
- calmer palette
- stronger CTA emphasis
- more breathing space

---

## 3. Updated Style DNA

Define the new design language.

Include:

- typography direction
- color palette
- spacing rhythm
- radius and shadow style

---

## 4. Component-Level Improvements

Describe changes to important components.

Examples:

- buttons
- cards
- forms
- navigation
- tables
- modals

---

## 5. Layout Improvements

Explain how the page or screen structure should evolve.

Examples:

- reorganized sections
- improved grid alignment
- simplified navigation

---

## 6. Expected Impact

Describe what improves.

Examples:

- clearer hierarchy
- faster scanning
- stronger conversion
- more professional feel

---

# Good Revamp Example

User:
“Bu dashboard amatör görünüyor.”

Good approach:

1. Identify issues
2. Define stronger hierarchy
3. Improve typography
4. Clean up components
5. Simplify colors

---

# Bad Revamp Behavior

Avoid:

- redesigning everything randomly
- ignoring the original product goal
- adding unnecessary visual effects
- making the interface more complex

Revamp should **improve clarity**, not create chaos.

---

# Success Criteria

A good Revamp Mode response should:

- clearly identify problems
- propose structured improvements
- create a stronger design system
- remain compatible with the product’s functionality
