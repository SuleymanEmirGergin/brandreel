# Conversion Mode

This mode is used when the interface’s primary goal is **conversion**.

Typical examples include:

- landing pages
- SaaS marketing sites
- waitlist pages
- product launch pages
- signup flows
- sales funnels

The goal is to maximize **clarity, persuasion, and action**.

---

# When To Use

Use this mode when the user asks for:

- a landing page
- SaaS marketing UI
- waitlist page
- product promotion interface
- signup or conversion flows

Example requests:

- “AI SaaS için landing page tasarla.”
- “Startup için waitlist page yap.”
- “SaaS marketing page UI öner.”
- “Ürün tanıtım sayfası tasarla.”

---

# Core Philosophy

Conversion-focused UI should guide users through a **clear narrative**.

A good conversion page answers these questions quickly:

1. What is this product?
2. Who is it for?
3. Why should I care?
4. Why should I trust it?
5. What should I do next?

The design must lead users toward the **primary action**.

---

# Conversion Principles

Strong conversion pages follow these rules.

## 1. Clarity First

The product value must be obvious within seconds.

Users should immediately understand:

- the product
- the benefit
- the action

Avoid vague messaging.

---

## 2. Strong Hero Section

The hero section should contain:

- clear headline
- supporting explanation
- primary CTA
- optional visual or illustration

The hero is the **first conversion opportunity**.

---

## 3. Visual Hierarchy

Important information should stand out.

Hierarchy should emphasize:

1. headline
2. product benefit
3. call to action
4. supporting proof

Avoid equal visual weight everywhere.

---

## 4. Social Proof

Trust signals improve conversion.

Examples:

- testimonials
- company logos
- user numbers
- ratings
- case studies

These elements reduce hesitation.

---

## 5. Benefit-Focused Sections

Explain the product through benefits.

Typical sections include:

- key features
- benefits
- use cases
- product screenshots
- workflow explanation

Focus on **value**, not just features.

---

## 6. CTA Strategy

The page should guide users toward one primary action.

Examples:

- Start free trial
- Join waitlist
- Book demo
- Get started

CTA buttons must be:

- visually prominent
- repeated strategically
- easy to access

---

## 7. Section Flow

Most effective landing pages follow this structure:

Hero  
→ Problem  
→ Solution  
→ Features  
→ Social Proof  
→ How It Works  
→ Pricing or Offer  
→ Final CTA

The goal is to gradually **increase confidence**.

---

# Layout Patterns

Common layout patterns include:

- hero with product preview
- feature cards
- split content sections
- testimonial blocks
- pricing tables
- FAQ sections

Spacing should provide breathing room between sections.

---

# Conversion Components

Key UI components for conversion pages:

- CTA buttons
- feature cards
- testimonials
- pricing tables
- product screenshots
- benefit lists
- FAQ accordions

These components should support **scanning behavior**.

---

# Output Structure

When using Conversion Mode, structure the response like this.

## 1. Conversion Goal

Explain what action the page should drive.

Examples:

- signups
- waitlist registrations
- demo bookings

---

## 2. Page Narrative

Explain the story flow of the page.

Example:

- introduction
- problem framing
- product value
- trust signals
- final call to action

---

## 3. Section Structure

List the recommended sections.

Example:

- hero
- features
- use cases
- testimonials
- pricing
- CTA

---

## 4. Component Strategy

Explain how UI components should support conversion.

Examples:

- strong CTA buttons
- clear feature cards
- visual product previews

---

## 5. Layout Guidance

Explain layout patterns and spacing rhythm.

---

## 6. Mobile Behavior

Describe how the landing page adapts to mobile screens.

Examples:

- stacked sections
- simplified navigation
- persistent CTA

---

# Anti-Patterns

Avoid:

- cluttered hero sections
- too many CTAs
- weak headline messaging
- excessive decorative elements
- unclear value proposition

A conversion page should **reduce friction**, not increase it.

---

# Success Criteria

A good Conversion Mode response should:

- clarify the product value
- guide users toward action
- use strong hierarchy
- build trust gradually
- remain visually clean and readable
