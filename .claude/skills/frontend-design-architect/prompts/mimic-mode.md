# Mimic Mode

This mode is used when the user wants a UI direction inspired by a reference link, screenshot, product, or existing interface.

The purpose is not to copy the reference literally, but to extract its design DNA and reinterpret it in a new product context.

---

# When To Use

Use this mode when the user says things like:

- “Şuna benzer bir tasarım yap.”
- “Bu sitenin hissini yakala.”
- “Stripe gibi ama birebir olmayan bir landing page istiyorum.”
- “Bu dashboard’ın stilini referans al.”

This mode is especially useful when the user values:

- visual taste
- premium feel
- stylistic similarity
- reference-based UI direction

---

# Core Goal

Your job is to identify what makes the reference _feel_ the way it does, then translate that into an original UI direction.

You are not reproducing the reference.

You are extracting:

- layout discipline
- typography personality
- color temperature
- spacing rhythm
- component weight
- CTA tone
- section density

Then you reapply those qualities in a way that fits the user’s product.

---

# What To Analyze

When a reference exists, inspect the following:

## 1. Overall Tone

What is the general vibe?

Possible examples:

- premium
- minimal
- technical
- modern
- editorial
- enterprise
- playful
- luxurious

---

## 2. Typography Personality

Look for:

- geometric or neutral sans-serif usage
- contrast between headings and body
- compact or spacious line-height
- strong or soft headline presence

Questions:

- Is it sharp and technical?
- Is it soft and premium?
- Is it editorial and expressive?

---

## 3. Color Logic

Analyze:

- palette temperature
- contrast strategy
- accent frequency
- primary CTA treatment
- muted background usage

Questions:

- Does the design rely on neutrals?
- Is there one strong accent color?
- Is the interface calm or high-energy?

---

## 4. Spacing Rhythm

Observe:

- section breathing room
- card padding
- density of content
- margin consistency

Questions:

- Is the design dense and efficient?
- Is it airy and premium?
- Does spacing help create calmness or speed?

---

## 5. Shape Language

Analyze:

- border radius
- corner sharpness
- component softness
- geometry consistency

Examples:

- sharp and business-like
- softly rounded and approachable
- nearly flat and minimal

---

## 6. Elevation and Surfaces

Analyze:

- shadow strength
- layered surfaces
- card separation
- flat vs elevated feel

Questions:

- Is the UI flat and quiet?
- Does it use subtle depth?
- Are surfaces heavily layered?

---

## 7. Component Weight

Study how heavy or light the components feel.

Check:

- button prominence
- input density
- card visual weight
- nav structure
- divider usage

Questions:

- Are buttons bold or subtle?
- Are cards heavy containers or light groupings?
- Is the UI component-rich or content-first?

---

## 8. Layout Discipline

Observe:

- section order
- column structure
- bento vs grid vs split layout
- navigation style
- hero composition

Important:
Do **not** copy the exact structure.  
Only identify the design discipline behind it.

---

# Anti-Copy Rule

Never reproduce the original:

- exact hero layout
- exact card arrangement
- exact section sequence
- exact navigation composition
- exact branded patterns

Instead:

- preserve the premium level
- preserve the rhythm
- preserve the visual intelligence
- preserve the design maturity

But create a new structure for the user’s context.

---

# Output Structure

When using Mimic Mode, use this response structure unless the user asks for something narrower.

## 1. Reference Style Read

Summarize the reference’s feel and design character.

## 2. Transferable Design DNA

List the abstract qualities worth keeping.

Examples:

- tight typography hierarchy
- cool neutral palette
- restrained shadows
- disciplined section spacing

## 3. Reinterpreted UI Direction

Explain how those qualities should be adapted to the user’s product.

## 4. Screen / Page Structure

Propose a fresh layout direction inspired by the reference but not copied.

## 5. Component Notes

Describe how buttons, cards, forms, nav, and surfaces should behave.

## 6. Responsive Notes

Explain how the same design DNA should behave on smaller screens.

---

# Good Behavior Example

User:
“Stripe benzeri ama daha sıcak bir B2B landing page istiyorum.”

Good response behavior:

- identify Stripe-like qualities
- keep the disciplined hierarchy
- soften the palette
- warm up the typography tone
- propose a new landing page structure

Bad behavior:

- recreate Stripe’s exact hero
- reuse the same visual sequence
- mimic the same component positions

---

# Response Style

Be specific.

Avoid vague phrases like:

- “modern vibe”
- “sleek feel”

Instead say:

- “Use compact headline typography with strong contrast against muted body text.”
- “Keep a cool-neutral base palette with a softer accent to avoid over-corporate stiffness.”

---

# Success Criteria

A strong Mimic Mode answer should:

- feel inspired by the reference
- not resemble a direct clone
- preserve quality level
- fit the new product context
- remain implementable
