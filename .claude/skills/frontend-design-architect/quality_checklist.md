# Quality Checklist

This checklist ensures that responses from the `frontend-design-architect` skill maintain high design quality, UX clarity, and implementation awareness.

Before finalizing any answer, the agent should mentally validate the following points.

---

# Product Alignment

Check that the design direction matches the user's real goal.

Questions to verify:

- Is the design aligned with the product's purpose?
- Does the UI help achieve the product goal?
- Is the platform considered correctly?

Example goals:

- conversion
- trust
- productivity
- exploration
- management
- content consumption

---

# Style Quality

Verify that the visual language is well defined.

Check:

- Is typography direction clear?
- Is the color palette coherent?
- Is spacing rhythm explained?
- Are radius and shadow decisions consistent?
- Is the contrast strategy understandable?

Avoid vague language.

Bad example:
“Modern design.”

Good example:
“Neutral palette with a single accent color and generous spacing.”

---

# Structural Quality

Ensure the layout hierarchy is understandable.

Check:

- Are the main sections defined?
- Is navigation clear?
- Can users scan the interface quickly?
- Is information hierarchy logical?

A strong UI should be **easy to scan within seconds**.

---

# Component Quality

Ensure the UI direction includes reusable components.

Check:

- Are core components identified?
- Are components reusable?
- Are component states considered where relevant?

Examples of core components:

- buttons
- inputs
- cards
- navigation elements
- tables
- modals
- empty states

Avoid designing only static layouts.

---

# UX Quality

Check if the UI improves usability.

Questions:

- Does the design reduce friction?
- Are key actions obvious?
- Is the CTA clear in conversion-focused pages?
- Is information density controlled in dashboards?

The UI must support **fast comprehension**.

---

# Responsive Quality

Ensure cross-device behavior is addressed.

Check:

- Is mobile behavior defined?
- Are tablet adaptations mentioned if relevant?
- Does the design language remain consistent across breakpoints?

A UI must remain usable on:

- mobile
- tablet
- desktop

---

# Implementation Awareness

If the user wants developer-friendly output, verify:

- Are tokens suggested where useful?
- Are components described clearly?
- Does the design align with common frontend stacks?

Example stacks:

- React
- Next.js
- Tailwind
- shadcn
- Expo
- React Native

Design advice should be **buildable**.

---

# Anti-Copy Safety

Check for accidental cloning.

Verify:

- Is the reference reinterpreted instead of copied?
- Are design principles extracted instead of layout duplication?

Always translate references into:

- style DNA
- design principles
- component behavior

---

# Response Discipline

Final check before answering:

- Is the response structured?
- Is it specific rather than generic?
- Is it focused on the user's request?
- Is unnecessary information avoided?

The best answers are:

- clear
- structured
- practical
- implementable
