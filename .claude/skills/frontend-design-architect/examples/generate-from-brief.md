# Example: Generate From Brief

## User Request

“AI destekli proje yönetim aracı için web app dashboard tasarla. Hedef his Linear + Stripe arası ama daha sıcak.”

---

## Expected Task Type

Generate

---

## Expected Mode

App UX Mode + Mimic Mode

---

## Expected Output Scope

The response should generate a new UI direction from the brief.

It should include:

- product context
- style DNA
- dashboard structure
- reusable components
- responsive notes

Optional:

- implementation-aware notes if useful

---

## Expected Response Structure

1. Design Summary
2. Style DNA
3. Information Architecture
4. Screen Structure
5. Component System
6. Responsive Notes
7. Optional Implementation Notes

---

## Good Response Characteristics

A strong answer should:

- define the product as a productivity-focused web app
- interpret “Linear + Stripe but warmer” as a style direction, not a clone request
- balance clarity with premium feel
- include dashboard primitives like sidebar, topbar, stat cards, task lists, filters, and detail views
- keep the structure highly scannable

---

## Example Good Output Direction

- Typography: compact, clean, technical but not cold
- Colors: cool-neutral base with warmer accent support
- Layout: left sidebar, lightweight topbar, modular dashboard sections
- Components: task table, project cards, activity panel, quick action bar
- Responsive: sidebar collapse on tablet, stacked cards on mobile

---

## Bad Response Characteristics

Avoid:

- copying Linear or Stripe layout directly
- giving only a mood description with no structure
- ignoring product workflow
- skipping component system thinking
