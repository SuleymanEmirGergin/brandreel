# Example: Redesign Existing UI

## User Request

“Bu ekran çok amatör duruyor. Premiumlaştır ama sade kalsın.”

---

## Expected Task Type

Revamp

---

## Expected Mode

Revamp Mode

---

## Expected Output Scope

The response should focus on improving the existing UI.

It should include:

- current weaknesses
- proposed visual direction
- component-level improvements
- hierarchy and spacing fixes
- expected outcome

It should not jump straight into full implementation unless requested.

---

## Expected Response Structure

1. Current UI Problems
2. New Visual Direction
3. Updated Style DNA
4. Component-Level Improvements
5. Layout Improvements
6. Expected Impact

---

## Good Response Characteristics

A strong answer should:

- identify what feels amateur
- explain which visual decisions weaken the product
- propose a cleaner, more premium direction without adding unnecessary decoration
- improve hierarchy, spacing, typography, and components
- preserve simplicity

---

## Example Good Output Direction

- Problems: weak hierarchy, inconsistent spacing, low-contrast components, generic card styling
- New Direction: calmer palette, stronger type scale, cleaner spacing rhythm, more refined surfaces
- Components: stronger buttons, lighter cards, simplified navigation, more intentional section spacing
- Impact: better scanability, stronger premium feel, clearer UI hierarchy

---

## Bad Response Characteristics

Avoid:

- “make it modern” style empty advice
- overdecorating with gradients, glows, and heavy shadows
- redesigning everything without preserving the original product goal
- giving vague criticism with no actionable next step
